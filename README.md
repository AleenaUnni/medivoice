# MediVoice 🎙️
### Voice-First Medical Intake Assistant

> Patients speak. Doctors arrive prepared.

Built for the **Amazon Nova Hackathon** using Amazon Nova 2 Sonic and Nova Lite on AWS Bedrock.

## 🏥 What is MediVoice?

MediVoice is a pre-visit patient intelligence platform that replaces paper forms with a natural voice conversation. Before their appointment, patients have a 3-minute spoken conversation with an AI assistant. By the time the doctor walks in, they already have a complete clinical brief — chief complaint, symptoms, pain scale, medications, allergies, red flags, and an AI-generated summary.

---

## ✨ Features

- 🎙️ **Voice Intake** — Real-time voice conversation powered by Amazon Nova 2 Sonic
- 🧠 **AI Report Generation** — Structured clinical brief generated by Amazon Nova Lite
- 👨‍⚕️ **Doctor Dashboard** — Patient sessions list with full clinical briefs
- 🔐 **Authentication** — Separate JWT-secured portals for patients and doctors
- 🖨️ **Print-Ready Reports** — Clinical briefs formatted for printing
- 💾 **Persistent Storage** — Patient data persists across server restarts
- 🌍 **Multilingual** — Supports English, Hindi, Malayalam, Spanish, Arabic

## 🏗️ Architecture
```
Patient Browser          Backend                    AWS Bedrock
─────────────           ─────────                  ───────────
Voice Input      →   WebSocket Server    →    Amazon Nova 2 Sonic
                      (port 8081)               (Real-time voice)

End Check-In     →   FastAPI Server      →    Amazon Nova Lite
                      (port 8000)               (Report generation)

Doctor Browser   ←   GET /sessions       ←    Structured JSON Report
```

## 🚀 Getting Started

### Prerequisites
- Python 3.12+
- Node.js 18+
- AWS Account with Bedrock access
- Amazon Nova 2 Sonic and Nova Lite enabled in us-east-1

### 1. Clone the repository
```bash
git clone https://github.com/yourusername/medivoice.git
cd medivoice
```

### 2. Backend Setup
```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Create `.env` file:
```
AWS_ACCESS_KEY_ID=your_access_key
AWS_SECRET_ACCESS_KEY=your_secret_key
AWS_DEFAULT_REGION=us-east-1
HOST=localhost
WS_PORT=8081
```

### 3. Frontend Setup
```bash
cd medivoice-ui
npm install
```

### 4. Run the Application

Terminal 1 — Nova Sonic WebSocket Server:
```bash
cd backend
source .venv/bin/activate
export $(cat .env | xargs)
python server.py
```

Terminal 2 — FastAPI Report Server:
```bash
cd backend
source .venv/bin/activate
uvicorn report_api:app --reload --port 8000
```

Terminal 3 — React Frontend:
```bash
cd medivoice-ui
npm start
```

### 5. Open the app
```
http://localhost:3000
```

## 📱 How It Works

1. **Patient logs in** → starts voice check-in
2. **Nova 2 Sonic** conducts the intake conversation
3. Patient answers questions about symptoms, pain, medications, allergies
4. Patient clicks **End Check-In**
5. **Nova Lite** analyzes the transcript and generates a structured clinical brief
6. **Doctor logs in** → sees patient in the sessions list
7. Doctor clicks **View Brief** → full clinical dashboard
